## 二、设计原理

### 2.1 为什么存在

JavaScript 生态拥有海量未类型化的库，TypeScript 必须与这些资产共存。互操作性机制的存在让渐进类型化成为可能，降低了采用 TS 的门槛。

### 2.2 权衡分析

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| .d.ts 声明 | 类型安全、IDE 支持 | 需要额外维护 | 第三方 JS 库 |
| JSDoc 注释 | 无需构建步骤 | 类型表达力有限 | 小型 JS 项目 |

### 2.3 与相关技术的对比

与纯 TS 项目对比：互操作降低了迁移门槛，但增加了类型维护成本。
