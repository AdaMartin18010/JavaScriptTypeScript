## 二、设计原理

### 2.1 为什么存在

JavaScript 基于原型而非类，但 ES6 class 语法提供了更 familiar 的 OOP 体验。对象与类的机制是组织复杂数据和行为的基石。

### 2.2 权衡分析

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 类继承 | 代码复用直观 | 紧耦合、脆弱基类 | IS-A 关系明确 |
| 对象组合 | 灵活、低耦合 | 更多样板代码 | 行为组装 |

### 2.3 与相关技术的对比

与经典 OOP 对比：JS 原型委托提供了更轻量的继承机制。
