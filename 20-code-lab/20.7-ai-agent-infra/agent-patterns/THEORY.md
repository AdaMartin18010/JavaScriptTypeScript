## 二、设计原理

### 2.1 为什么存在

LLM 的自主能力需要通过结构化的模式来引导。Agent 模式将推理、规划和工具调用组织为可复用的架构，提升 AI 系统的可控性和可靠性。

### 2.2 权衡分析

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| ReAct | 推理透明 | 延迟高 | 复杂决策 |
| 直接调用 | 速度快 | 不可解释 | 简单任务 |

### 2.3 与相关技术的对比

与硬编码流程对比：模式提升适应性，硬编码更确定和高效。

### 3.1 从理论到代码

本模块的代码示例将上述理论概念映射为可运行的实现。通过实际编码练习，可以验证对 Agent 模式 核心机制的理解，并观察不同实现选择带来的行为差异。

### 3.2 常见误区

| 误区 | 正确理解 |
|------|---------|
| Agent 模式保证正确结果 | 模式提升可控性，不消除模型幻觉 |
| 越复杂的模式越好 | 简单任务使用简单模式降低延迟和成本 |
