# AI Agent 实验室 — 理论基础

## 1. Agent 的定义

AI Agent 是一个能够**感知环境、做出决策并执行行动**的自主系统。与简单的 LLM 调用不同，Agent 具备：

- **目标导向**: 围绕特定任务目标运行
- **工具使用**: 能调用外部 API、数据库、搜索引擎
- **记忆能力**: 维护上下文和历史状态
- **规划能力**: 将复杂任务分解为子任务

## 2. Agent 架构模式

### 2.1 ReAct (Reasoning + Acting)

```
思考 → 行动（调用工具）→ 观察（获取结果）→ 思考 → ...
```

这种模式使 Agent 能够逐步解决问题，每一步都基于前一步的结果。

### 2.2 Plan-and-Solve

1. **规划阶段**: LLM 生成完整的执行计划（步骤列表）
2. **执行阶段**: 按顺序执行每个步骤，收集结果
3. **验证阶段**: 检查结果是否满足目标，必要时重新规划

### 2.3 Multi-Agent 协作

- **分工模式**: 不同 Agent 负责不同子任务（研究→写作→审校）
- **辩论模式**: 多个 Agent 对同一问题提出不同观点，综合最优解
- **层级模式**: 主管 Agent 分配任务，工作 Agent 执行并汇报

## 3. MCP (Model Context Protocol)

MCP 是 Anthropic 提出的开放标准，定义了 LLM 与外部工具/资源的标准接口：

- **Server**: 暴露工具和资源（如文件系统、数据库、API）
- **Client**: 连接到 Server，将工具列表提供给 LLM
- **Protocol**: JSON-RPC 2.0 基于的通信协议

MCP 解决了每个 LLM 框架都需要单独适配每个工具的问题。

## 4. A2A (Agent-to-Agent) 协议

Google 提出的 A2A 协议用于不同 Agent 之间的协作：

- **Agent Card**: JSON 描述 Agent 的能力、端点、认证方式
- **Task**: 工作单元，具有提交→工作中→输入请求→完成/失败的生命周期
- **Artifact**: Agent 产出的结构化结果

## 5. 记忆管理

| 记忆类型 | 存储 | 生命周期 | 用途 |
|---------|------|---------|------|
| **工作记忆** | 上下文窗口 | 单次对话 | 当前任务上下文 |
| **短期记忆** | 向量数据库 | 会话级 | 相关历史信息检索 |
| **长期记忆** | 图数据库/关系库 | 持久 | 用户偏好、知识积累 |
| **episodic** | 日志系统 | 持久 | 经验回放、反思学习 |
