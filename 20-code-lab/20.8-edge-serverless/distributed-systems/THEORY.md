# 分布式系统 — 理论基础

## 1. CAP 定理

分布式系统最多同时满足以下两项：

- **C**onsistency（一致性）: 所有节点看到相同数据
- **A**vailability（可用性）: 每个请求都得到响应
- **P**artition Tolerance（分区容错性）: 网络分区时系统仍能运行

实际系统中 P 必须满足，因此只能在 C 和 A 之间权衡：

- **CP 系统**: 牺牲可用性（ZooKeeper、etcd、HBase）
- **AP 系统**: 牺牲一致性（Cassandra、DynamoDB、Couchbase）

## 2. BASE 理论

对 CAP 中 AP 路径的细化：

- **B**asically **A**vailable: 基本可用，允许部分故障
- **S**oft state: 软状态，允许中间不一致状态
- **E**ventually consistent: 最终一致性，保证在一定时间后达成一致

## 3. 一致性模型谱系

从强到弱：

```
线性一致性 → 顺序一致性 → 因果一致性 → 会话一致性 → 最终一致性
```

## 4. 故障模型

| 故障类型 | 描述 | 处理策略 |
|---------|------|---------|
| **崩溃停止** | 节点宕机不再恢复 | 心跳检测 + 副本切换 |
| **崩溃恢复** | 节点宕机后可恢复 | 持久化日志 + 状态恢复 |
| **拜占庭故障** | 节点任意行为（包括恶意） | PBFT、区块链共识 |
| **网络分区** | 节点间通信中断 | 分区感知 + 仲裁机制 |
