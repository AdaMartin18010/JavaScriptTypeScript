## 二、设计原理

### 2.1 为什么存在

传统中心化架构难以满足全球用户的低延迟需求。边缘优先模式将计算和数据推向离用户最近的节点，是新一代 Web 应用的核心架构思想。

### 2.2 权衡分析

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 边缘渲染 | 低延迟 | 计算受限 | 全球用户 |
| 中心渲染 | 算力充沛 | 网络延迟 | 区域用户 |

### 2.3 与相关技术的对比

与云原生对比：边缘降低延迟，云原生计算资源更充沛。
