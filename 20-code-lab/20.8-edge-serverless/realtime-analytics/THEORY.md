# 实时分析 — 理论基础

## 1. 流处理架构

```
数据源 → 消息队列（Kafka/Pulsar）→ 流处理引擎（Flink/Spark Streaming）→ 实时仪表盘
```

## 2. 窗口操作

流数据的时间聚合：

- **滚动窗口**: 固定大小、不重叠（每 5 分钟）
- **滑动窗口**: 固定大小、可重叠（每 1 分钟统计过去 5 分钟）
- **会话窗口**: 按活动间隙划分（用户 30 分钟无操作视为会话结束）
- **全局窗口**: 整个流一个窗口（用于全局聚合）

## 3.  exactly-once 语义

确保每条数据只被处理一次：

- **幂等写入**: 相同数据多次写入结果一致
- **事务性提交**: 偏移量与结果原子提交
- **Checkpoints**: 定期保存处理状态
