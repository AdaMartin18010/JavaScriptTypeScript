# 混沌工程 — 理论基础

## 1. 混沌工程原则

Netflix 提出的生产环境韧性验证方法：

1. 建立稳态假设（系统正常行为的度量）
2. 引入真实世界的故障事件
3. 验证稳态是否保持
4. 若稳态被破坏，找到并修复问题

## 2. 故障注入类型

| 类型 | 工具 | 影响 |
|------|------|------|
| **网络延迟** | Toxiproxy、Chaos Mesh | 模拟慢网络 |
| **服务宕机** | Chaos Monkey | 随机终止实例 |
| **CPU 满载** | stress-ng | 资源竞争 |
| **内存耗尽** | Linux cgroups | OOM 场景 |
| **磁盘故障** | disk-fill | 存储压力 |
| **时钟跳跃** | libfaketime | 时间敏感系统 |

## 3. 游戏日（Game Day）

有组织的混沌实验：

- 提前通知相关团队
- 设定明确的实验范围和回滚条件
- 实时监控业务指标
- 实验结束后复盘总结
