# 技术雷达模板

> 团队技术雷达的标准化模板，用于定期评估技术栈健康度。

---

## 四象限定义

| 象限 | 说明 |
|------|------|
| **Adopt（采用）** | 已验证，团队应默认使用 |
| **Trial（试验）** | 有前景，可在非核心项目尝试 |
| **Assess（评估）** | 值得关注，需持续跟踪 |
| **Hold（暂缓）** | 已过时或有明显缺陷，避免新项目使用 |

## 雷达分类

| 分类 | 示例技术 |
|------|---------|
| **语言与框架** | React, Vue, Svelte, Next.js |
| **工具链** | Vite, Turborepo, Biome, pnpm |
| **平台与基础设施** | Vercel, Cloudflare, Railway, Kubernetes |
| **数据与 AI** | PostgreSQL, TanStack Query, OpenAI, MCP |

## 使用方式

1. 每季度组织技术雷达评审会
2. 每个技术由实际使用者提供评估依据
3. 记录技术迁移决策（从 Trial → Adopt，或 Adopt → Hold）
4. 将雷达结果写入团队知识库
