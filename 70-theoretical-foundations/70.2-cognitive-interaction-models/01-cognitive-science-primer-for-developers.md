## 1. 工作记忆

### 1.1 Miller 的 7±2

George Miller (1956) 的经典研究表明，人类工作记忆的容量约为 **7±2 个信息块**（Chunks）。然而，后续研究（Cowan, 2001）更精确地估计为 **4±1 个块**。

**对编程的启示**：
- 函数的参数数量应控制在 4 个以内
- 嵌套深度应限制在 3-4 层以内
- 一个代码块中的变量数量应有限制

### 1.2 Baddeley 的工作记忆模型

Baddeley (2007) 提出了工作记忆的四个子系统：

| 子系统 | 功能 | 编程对应 |
|--------|------|---------|
| **语音环路** | 存储语言信息 | 变量名、API 名称 |
| **视觉空间画板** | 存储视觉信息 | 代码结构、缩进层次 |
| **情景缓冲器** | 整合多模态信息 | 代码与文档的关联 |
| **中央执行** | 注意力控制和协调 | 理解复杂逻辑 |

## 2. 长期记忆

### 2.1 陈述性记忆 vs 程序性记忆

| 类型 | 内容 | 编程对应 |
|------|------|---------|
| **陈述性记忆**（语义）| 概念、事实 | 语言语法、API 文档 |
| **陈述性记忆**（情景）| 个人经历 | 调试经验、项目经历 |
| **程序性记忆** | 技能和习惯 | 打字、代码模式识别 |

### 2.2 编码、巩固、提取

```
编码（Encoding）
    ↓
巩固（Consolidation）—— 睡眠期间加强
    ↓
提取（Retrieval）—— 回忆和应用
```

**对编程教育的启示**：间隔重复（Spaced Repetition）比集中学习更有效。

## 3. 注意力的认知模型

### 3.1 三种注意力

| 类型 | 定义 | 编程场景 |
|------|------|---------|
| **选择性注意** | 聚焦于特定刺激 | 阅读关键代码行 |
| **分散注意** | 同时处理多个任务 | 边写代码边查看文档 |
| **持续注意** | 长时间维持专注 | 长时间调试 |

### 3.2 注意力资源理论

Kahneman (1973) 的注意力资源理论：注意力是有限的认知资源。

**对 IDE 设计的启示**：
- 语法高亮帮助选择性注意
- 代码折叠减少视觉干扰
- 错误提示不应过于突兀（避免注意力劫持）

## 4. 认知负荷理论（CLT）

### 4.1 三种认知负荷

Sweller (2011) 提出了认知负荷的三种类型：

| 类型 | 定义 | 编程示例 |
|------|------|---------|
| **内在负荷** | 任务本身的复杂度 | 理解递归算法 |
| **外在负荷** | 信息呈现方式 | 混乱的代码格式 |
| **相关负荷** | 促进学习的处理 | 类型推断帮助理解 |

### 4.2 减少外在负荷的策略

```typescript
// 高外在负荷：回调地狱
doA(() => doB(() => doC(() => doD(result => {
  console.log(result);
}))));

// 低外在负荷：async/await
const result = await doD(await doC(await doB(await doA())));
console.log(result);
```

## 5. 心智模型（Mental Models）

### 5.1 Johnson-Laird 的定义

心智模型是人们对现实世界的心理表征，用于推理和预测。

### 5.2 编程中的心智模型

| 概念 | 新手心智模型 | 专家心智模型 |
|------|-------------|-------------|
| 变量 | 盒子里的值 | 绑定到值的名称 |
| 引用 | 指向盒子的箭头 | 内存地址的抽象 |
| 闭包 | "魔法" | 环境捕获的显式机制 |
| 原型链 | 混乱的继承 | 委托链的精确理解 |

## 6. 具身认知（Embodied Cognition）

### 6.1 身体经验影响抽象思维

Lakoff & Johnson (1999) 提出：抽象概念通过**隐喻**建立在身体经验之上。

**编程中的具身认知**：
- "向上" = 更抽象（"高层 API"）
- "向下" = 更具体（"底层实现"）
- "向前" = 执行（"推进任务"）
- "向后" = 撤销（"回滚更改"）

### 6.2 为什么某些范式"感觉更自然"

| 范式 | 具身隐喻 | 自然性来源 |
|------|---------|-----------|
| 命令式 | "做事"的隐喻 | 日常动作经验 |
| 函数式 | "流水线"隐喻 | 工厂/厨房经验 |
| 面向对象 | "物体交互"隐喻 | 物理世界经验 |
| 响应式 | "因果链"隐喻 | 物理因果经验 |

## 7. 认知科学在编程中的应用框架

### 7.1 分析框架

```
代码/工具分析
  ├── 工作记忆负荷：需要同时记住多少信息？
  ├── 认知负荷类型：内在？外在？相关？
  ├── 注意力分配：注意力被引导到哪里？
  ├── 心智模型匹配：与开发者的心智模型一致吗？
  └── 具身认知：使用什么隐喻？自然吗？
```

### 7.2 设计原则

1. **减少工作记忆负荷**：分块信息，提供外部记忆辅助
2. **优化认知负荷**：减少外在负荷，增加相关负荷
3. **引导注意力**：使用视觉层次和渐进披露
4. **匹配心智模型**：使用开发者熟悉的隐喻
5. **利用具身认知**：使用自然的方向性和动作隐喻
