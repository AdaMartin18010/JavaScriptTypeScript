# Awesome JavaScript/TypeScript Ecosystem - 收录评分标准

本文档定义了项目收录 JavaScript/TypeScript 库的评分标准和收录流程。

---

## 收录标准

### 必要条件（必须满足）

| 条件 | 说明 | 检查方式 |
|------|------|----------|
| 1. GitHub Stars >= 1000 | 库需要有一定的社区认可度 | GitHub API 或页面查看 |
| 2. 最近6个月内有提交 | 确保库仍在维护中，非废弃状态 | 查看最后提交时间 |
| 3. 有 README 文档 | 必须有基本的项目介绍和使用说明 | 检查仓库根目录 |
| 4. 有 License | 必须有明确的开源许可证 | 检查 LICENSE 文件 |

> ⚠️ **注意**：以上4个条件必须**全部满足**，否则不予收录。

---

## TypeScript 支持评级

TypeScript 支持程度是本项目收录的重要考量因素。

| 评级 | 等级 | 定义 | 图标 |
|------|------|------|------|
| **Level 1** | 原生 TypeScript 编写 | 源代码使用 TypeScript 编写，提供原生类型支持 | 🟢 |
| **Level 2** | 官方类型定义 | 通过 `@types/xxx` 或内置 `.d.ts` 文件提供官方类型支持 | 🟡 |
| **Level 3** | 社区类型定义 | 仅有社区维护的类型定义，或类型支持较弱 | 🔴 |

### 评级说明

- 🟢 **Level 1 - 原生 TypeScript**
  - 源代码使用 TypeScript 编写
  - 无需额外安装类型定义包
  - 类型定义与代码同步更新

- 🟡 **Level 2 - 官方类型定义**
  - JavaScript 编写，但提供官方维护的类型定义
  - 类型定义在 DefinitelyTyped 有官方维护，或仓库内置 `.d.ts` 文件
  - 类型定义与库版本保持同步

- 🔴 **Level 3 - 社区类型定义**
  - 仅有社区维护的 `@types/xxx` 包，且非官方维护
  - 类型定义可能滞后于库版本
  - 类型支持不完整或存在已知问题

## 维护状态评级

维护状态反映了库的生命周期活跃度。

| 评级 | 状态 | 定义 | 图标 |
|------|------|------|------|
| **活跃** | 最近1个月有提交 | 项目正在积极开发中 | 🟢 |
| **维护** | 最近3个月有提交 | 项目处于维护模式，修复 bug 和接受 PR | 🟡 |
| **低频** | 最近6个月有提交 | 项目更新较少，但仍有人维护 | 🔴 |
| **归档** | 超过6个月无提交 | 项目可能已废弃或归档 | ⚫ |

> ⚠️ **注意**：**归档**状态的库原则上不予收录，除非在特定领域无可替代。

## 流行度评级

基于 GitHub Stars 数量的流行度评估。

| 评级 | Stars 范围 | 图标 |
|------|------------|------|
| **高** | > 10,000 | ⭐⭐⭐ |
| **中** | 1,000 - 10,000 | ⭐⭐ |
| **低** | < 1,000 | ⭐ |

> ℹ️ **说明**：收录最低门槛为 ⭐（< 1k Stars 的库不予收录）。

## 质量指标

以下指标用于综合评估库的质量，作为收录的参考依据。

### 1. 测试覆盖率

| 等级 | 覆盖率 | 说明 |
|------|--------|------|
| 优秀 | > 80% | 有完善的测试套件 |
| 良好 | 50% - 80% | 有基本测试覆盖 |
| 一般 | < 50% | 测试覆盖不足 |
| 未知 | - | 未公开测试覆盖率 |

### 2. 文档完整度

| 等级 | 标准 |
|------|------|
| 优秀 | 有详细文档网站、API 文档、示例代码、迁移指南 |
| 良好 | README 完整，包含安装、使用、配置说明 |
| 一般 | 仅有基本 README |
| 较差 | 文档缺失或严重不足 |

### 3. TypeScript 严格模式支持

| 等级 | 说明 |
|------|------|
| 完全支持 | 可在 `strict: true` 模式下无错误使用 |
| 部分支持 | 基本可用，但可能存在 `any` 类型或类型漏洞 |
| 不支持 | 类型定义存在问题，无法正常使用 |

### 4. 社区活跃度

通过 Issues 和 PR 的响应情况评估：

| 等级 | 标准 |
|------|------|
| 活跃 | Issue 平均响应时间 < 7 天，PR 定期合并 |
| 一般 | Issue 平均响应时间 7-30 天 |
| 冷淡 | Issue 和 PR 长期无响应 |

```
┌─────────────────┐
│   提交收录申请   │
└────────┬────────┘
         ▼
┌─────────────────┐

│ (4项必须全满足) │              ▼
└────────┬────────┘        ┌─────────┐
         │满足              │  拒绝   │
         ▼                  └─────────┘
┌─────────────────┐
│ 评估 TypeScript │
│   支持等级      │
└────────┬────────┘
         ▼
┌─────────────────┐
│ 评估维护状态    │
└────────┬────────┘
         ▼

         ▼

         │达到门槛           │  拒绝   │

```

## 特殊收录情况

以下情况可作为特殊收录的参考：

1. **领域唯一**：在特定领域没有更好的替代方案，即使 Stars < 1000
2. **新兴潜力**：新出现的库，增长势头良好（近3个月 Stars 增长 > 50%）
3. **大厂背书**：由知名公司或组织维护，质量有保障
4. **历史价值**：在 JS/TS 生态发展历史上有重要地位的库

---

## 更新与维护

- 本标准每季度评审一次
- 已收录的库如不满足必要条件，将标记为 `⚠️ 待观察`
- 连续两个评审周期不满足条件的库将被移除
