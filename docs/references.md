## 🏆 学到的最佳实践

### 1. 分类结构

```
清晰的层级结构示例：
├── 官方资源 (Official Resources)
├── 快速开始 (Getting Started)
│   ├── 模板 (Templates)
│   │   ├── Vue
│   │   ├── React
│   │   └── Svelte
│   └── 脚手架工具
├── 插件/工具 (Plugins/Tools)
│   ├── 框架无关
│   ├── Vue 生态
│   └── React 生态
├── 集成方案 (Integrations)
└── 社区资源 (Community)
```

**要点：**

- 使用 Table of Contents 便于导航
- 按技术栈或用途分层分类
- 热门项目可以单独归类

### 2. 收录标准

**质量筛选原则：**

- ✅ 项目成熟稳定（至少存在数月）
- ✅ 有一定 Star 数或社区认可度
- ✅ 持续维护（近期有更新）
- ✅ 文档完整
- ❌ 避免重复收录
- ❌ 避免推广性质的项目

**Awesome JavaScript 的贡献检查清单：**

```markdown
- [ ] 已阅读并理解贡献指南
- [ ] 将新资源添加到对应分类末尾
- [ ] 该资源已存在一段时间且持续维护
- [ ] 该资源足够流行，GitHub 上至少有数百 Stars
```

### 3. 自动化维护

**推荐工具：**

- **awesome-lint**: 检查列表格式规范
- **GitHub Actions**: 自动检测死链
- **Renovate**: 自动更新依赖
- **Stargazed**: 自动生成 Awesome 列表

**CI/CD 流程：**

```yaml
# 示例：链接检查
- name: Check Links
  uses: lycheeverse/lychee-action@v1
  with:
    args: --timeout 60 README.md
```

### 4. 文档规范

**格式约定：**

- 项目名称使用链接格式
- 每个项目后附简短描述
- 使用 Badge 标识版本/状态
- 提供返回目录的快捷链接

**示例格式：**

```markdown
- [project-name](link) - 简短描述，说明用途和特点。
- [another-project](link) ★1k - 带星数标记的流行项目。
```

### 5. 社区运营

**维护策略：**

- 制定清晰的贡献指南 (CONTRIBUTING.md)
- 使用 Issue 模板管理收录请求
- 定期清理过时项目
- 对贡献者进行致谢
