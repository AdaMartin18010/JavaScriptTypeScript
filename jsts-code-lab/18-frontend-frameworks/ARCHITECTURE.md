# 前端框架 — 架构设计

## 1. 架构概述

本模块实现了主流前端框架的核心机制简化版，包括响应式系统、虚拟 DOM Diff、组件生命周期管理和状态管理。通过 TypeScript 代码揭示框架背后的设计原理。

## 2. 核心组件

### 2.1 响应式引擎

- **Signal**: 细粒度的响应式原语，订阅-发布模式
- **Effect**: 副作用追踪器，自动收集依赖并重新执行
- **Computed**: 派生信号，缓存计算结果

### 2.2 虚拟 DOM 系统

- **VNode**: 虚拟节点抽象（标签、属性、子节点）
- **Diff 算法**: 同层比较，Key 优化列表更新
- **Patch 引擎**: 将差异应用到真实 DOM

### 2.3 组件系统

- **Component**: 函数式组件封装
- **Props/State**: 输入属性与内部状态管理
- **Lifecycle Hooks**: 挂载、更新、卸载钩子

## 3. 数据流

```
状态变更 → Signal 通知 → Effect 调度 → VNode 重新渲染 → Diff → Patch DOM
```

## 4. 技术决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 响应式粒度 | Signal 级别 | 最小化重新计算范围 |
| Diff 策略 | 同层 + Key | 平衡性能与实现复杂度 |
| 渲染触发 | 微任务批处理 | 减少 DOM 操作次数 |

## 5. 质量属性

- **性能**: 细粒度更新避免全量渲染
- **可预测性**: 单向数据流确保状态变化可追溯
- **可组合性**: 组件嵌套和逻辑复用
