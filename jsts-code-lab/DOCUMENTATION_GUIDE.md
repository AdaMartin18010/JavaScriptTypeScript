# 文档编写指南

## 文档结构标准

每个模块应包含以下文档文件：

### 1. README.md - 模块总览

```markdown
# 模块名称

## 理论背景
[理论基础、核心概念、历史发展]

## 设计原理
[为什么这样设计、解决了什么问题]

## 架构论证
[架构决策、取舍分析、最佳实践]

## 核心实现
[关键代码解析、设计模式应用]

## 使用场景
[适用场景、不适用场景、示例]

## 性能特征
[时间复杂度、空间复杂度、瓶颈分析]

## 扩展阅读
[相关论文、参考书籍、进一步学习资源]
```

### 2. ARCHITECTURE.md - 架构设计文档

### 3. THEORY.md - 理论基础文档（如需要）

---

## 文档质量检查清单

- [ ] 是否解释了"为什么"而不仅仅是"怎么做"
- [ ] 是否包含设计决策的权衡分析
- [ ] 是否有性能特征的理论分析
- [ ] 是否引用相关学术资源
- [ ] 是否包含可视化图表（架构图、流程图）
- [ ] 是否有实际应用场景说明
