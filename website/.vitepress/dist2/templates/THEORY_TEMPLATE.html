THEORY.md 统一模板 ​

本文档模板用于规范所有 THEORY.md 文件的格式，确保文档质量的一致性和可维护性。

文档结构 ​

markdown
# [主题名称]理论基础

## 1. 简介

简要介绍本主题的核心概念、研究背景和重要性。

## 2. 理论基础

### 2.1 核心概念定义
- 关键术语解释
- 概念之间的关系

### 2.2 形式化定义（如适用）
使用数学符号、集合论、逻辑表达式等形式化描述核心概念。

### 2.3 理论论证
- 定理/引理陈述
- 证明过程或证明概要
- 复杂度分析

## 3. 实现原理

### 3.1 工作机制
详细说明技术的内部工作原理。

### 3.2 算法描述
使用伪代码或步骤描述核心算法。

### 3.3 性能特征
- 时间复杂度
- 空间复杂度
- 实际性能影响因素

## 4. 应用场景

### 4.1 适用条件
明确说明技术在什么情况下适合使用。

### 4.2 使用示例
提供典型应用场景和代码示例。

### 4.3 最佳实践
- 推荐做法
- 常见陷阱

## 5. 相关理论

### 5.1 前置知识
学习本主题前需要掌握的基础理论。

### 5.2 进阶主题
与本主题相关的更深入理论。

### 5.3 相关模式/算法
相关联的设计模式、算法或架构。

## 6. 参考文献

### 6.1 经典著作
1. 作者. (年份). *书名*. 出版社.

### 6.2 学术论文
1. 作者. (年份). "论文标题". 期刊/会议名.

### 6.3 在线资源
- [资源名称](#) - 简要描述

写作规范 ​

数学符号使用 ​

  • 使用 LaTeX 风格的行内公式：$O(n \log n)$
  • 复杂公式使用代码块展示
  • 保持一致的定义风格

图表规范 ​

  • 使用 ASCII 艺术或 Mermaid 语法
  • 确保图表在纯文本中可读
  • 为每个图表添加简要说明

代码示例 ​

  • 使用 TypeScript 作为主要语言
  • 代码块标注语言类型
  • 关键代码行添加注释

引用规范 ​

  • 书籍：作者. (年份). 书名
  • 论文：作者. (年份). "标题"
  • 网页：标题
