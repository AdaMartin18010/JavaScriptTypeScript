## 推荐学习路径

### 路径一：全栈开发者

```
1. 语言核心 → TypeScript 语言全景 / TS 高级模式
2. 前端框架 → React / Vue / Svelte 选型
3. 构建工具 → Vite / Rspack / Webpack 对比
4. 后端开发 → API 设计 / 后端框架 / 数据库
5. 部署运维 → CI/CD / 部署平台 / 可观测性
6. 前沿技术 → AI-Native / Edge / WebAssembly
```

### 路径二：前端专家

```
1. 前端框架 → 框架对比矩阵 / SSR 元框架
2. 状态管理 → Zustand / Jotai / Signals
3. 性能优化 → Web Vitals / 构建优化
4. UI 组件 → shadcn/ui / Tailwind CSS
5. 测试策略 → Vitest / Playwright
6. 跨平台 → 移动端 / 桌面端 / WebAssembly
```

### 路径三：架构师

```
1. 系统设计 → 微服务 / Edge-First / API 设计
2. 性能工程 → 性能矩阵 / 监控工具
3. 数据库 → 数据库选型 / 迁移策略
4. 安全 → 安全合规 / 供应链安全
```

### 路径四：后端开发者

专为从其他后端语言（Python/Java/Go/Rust）转入 Node.js 生态，或希望深耕服务端 JS/TS 的开发者设计：

```
1. 运行时选择 → Node.js LTS 特性 / Deno 安全模型 / Bun 性能基准
2. 语言加固 → TypeScript 严格模式 / Zod 运行时校验 / tRPC 类型安全
```

```
1. 基础接入 → Vercel AI SDK / OpenAI SDK / Anthropic SDK 快速上手
```

## 常用工具速查

### TypeScript Playground

在线实验 TypeScript 类型系统的最佳工具：

```
https://www.typescriptlang.org/play
```

- 切换 TS 版本（支持 Nightly）
- 查看编译后的 JS 输出
- 分享链接给同事讨论类型问题
- 启用严格模式选项测试边界情况

### Vite 常用命令

```bash
# 创建项目
npm create vite@latest my-app -- --template react-ts
npm create vite@latest my-app -- --template vue-ts
npm create vite@latest my-app -- --template svelte-ts

# 开发服务器
npm run dev

# 生产构建
npm run build

# 预览生产构建
npm run preview
```

### npm / pnpm / yarn 速查

```bash
# 初始化项目
npm init -y

# 安装依赖
npm install <package>
npm install -D <dev-package>    # 开发依赖

# 运行脚本
npm run <script>
npx <package>                  # 临时执行

# pnpm 高效替代
pnpm install                   # 更快的安装速度
pnpm add <package>             # 添加依赖
pnpm dlx <package>             # 等效 npx

# 安全检查
npm audit
npm audit fix
```

### Node.js 版本管理

```bash
# 使用 nvm（Windows 推荐 nvm-windows）
nvm install 22
nvm use 22
nvm list

# 使用 fnm（跨平台，更快）
fnm install 22
fnm use 22
fnm default 22
```

### 调试命令

```bash
# Node.js 调试
node --inspect-brk index.js

# TypeScript 直接运行（ts-node / tsx）
npx tsx watch src/index.ts

# 性能分析
node --prof index.js
node --prof-process isolate-*.log > profile.txt
```

## 常见问题 FAQ

### Q1: 本站内容是否免费？可以用于商业项目吗？

**A**: 所有内容基于 MIT 协议开源，免费阅读、引用和用于商业项目。引用时建议注明出处链接。

### Q2: 内容多久更新一次？如何知道某个页面是否过时？

**A**: 每个页面底部标注「最后更新」日期。技术分类每月审核，对比矩阵每季度更新数据，深度指南按技术演进更新。详见下文「数据更新日历」。

### Q3: 我是初学者，内容太多不知从何开始？

**A**: 请遵循「路径七：学生 / 初学者」，从 🌱 入门标识的内容开始。建议先完成「语言基础导览」和「开发环境配置指南」，再动手做第一个项目。

### Q4: 对比矩阵中的数据是如何评估的？

**A**: 矩阵评分基于多维度综合评估：官方文档完整性、GitHub Stars / 周下载量、社区活跃度（Issue 响应时间、最近提交时间）、基准测试数据、生产环境采用案例。每项评分附带来源链接供核实。

### Q5: 某些前沿工具（如 🔥 标识）是否适合生产环境？

**A**: 🔥 前沿内容通常标记实验性工具的「当前状态」。生产环境采用前，请查看该工具页面的「生产就绪度评估」小节，确认其版本号、测试覆盖率和已知问题。

### Q6: 如何报告错误或提交改进建议？

**A**: 小错误可直接通过页面底部的「编辑此页」链接提交 PR；复杂建议请在 GitHub Discussions 发起讨论。详见 [贡献指南](./contributing)。

### Q7: 是否有英语版本？

**A**: 英语内容在 `/en/` 路径下维护，当前覆盖核心分类和主要矩阵。多语言完整翻译正在进行中，欢迎参与贡献。

### Q8: 代码实验室的示例如何本地运行？

**A**: 每个代码块右上角提供「复制」按钮。对于需要完整项目环境的示例，页面底部提供 StackBlitz / CodeSandbox 在线运行链接，或 GitHub 仓库的对应目录路径。

### Q9: 为什么我看到的对比矩阵数据和 npm 下载量不一致？

**A**: 矩阵数据基于固定时间点的快照（标注于矩阵底部），与实时 npm 数据可能存在时间差。趋势方向比绝对数字更有参考价值。每季度更新时会重新拉取最新数据。

### Q10: 如何获取本站内容的离线副本？

**A**: 方式一：克隆 GitHub 仓库获取完整 Markdown 源文件；方式二：使用浏览器「另存为 PDF」功能导出单页；方式三：订阅 RSS 获取增量更新。本站暂无官方电子书发布计划。

### Q11: 不同学习路径之间可以交叉吗？

**A**: 完全可以。路径是建议而非约束。例如，全栈开发者可以在掌握基础后切入「路径五：AI 开发者」学习 AI SDK 集成；后端开发者可以参考「路径二」了解前端框架以更好地协作。

### Q12: 如何跟踪本站依赖的外部工具版本变化？

**A**: 关注页面中的「版本兼容性」标注区域，以及对比矩阵中的「维护状态」列（活跃 / 维护中 / 停止维护）。重大版本变化（如 React 19、TypeScript 6）会在对应指南顶部添加迁移横幅。
