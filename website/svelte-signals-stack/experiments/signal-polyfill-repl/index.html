🧪 Experiment 1: Basic Counter

TC39 Signals

const count = new Signal.State(0); const doubled = new Signal.Computed(() => count.get() * 2); effect(() => console.log(count.get(), doubled.get())); count.set(1); // logs: 1 2

Svelte 5 Equivalent

let count = $state(0); let doubled = $derived(count * 2); $effect(() => console.log(count, doubled)); count++; // logs: 1 2

🧪 Experiment 2: Watcher & Batch

Manual Watcher (low-level API)

const a = new Signal.State(1); const b = new Signal.State(2); const sum = new Signal.Computed(() => a.get() + b.get()); const w = new Signal.subtle.Watcher(() => { console.log('Changed!', sum.get()); }); w.watch(sum); a.set(5); // Watcher notified, sum = 7
Click "Run" to see output...

// TC39: No built-in batching // Frameworks implement their own // Svelte 5: Automatic batching via flushSync // Multiple updates in one event → single effect run

📋 Semantic Equivalence Checklist

  • ✅ Signal.State ↔ $state : Writable reactive source
  • ✅ Signal.Computed ↔ $derived : Lazy evaluation, caching, no glitches
  • ⚠️ Watcher ↔ $effect : Watcher is lower-level primitive; $effect includes scheduler
  • ✅ untrack ↔ untrack : Same semantics (skip dependency registration)
  • ⚠️ Batch : TC39 has no built-in batching; Svelte has flushSync + microtask queue
