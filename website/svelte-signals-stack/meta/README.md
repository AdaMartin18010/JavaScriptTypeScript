# 支持索引与元数据

> **定位**: Svelte 5 Signals 编译器生态专题的支持文件集合
> **用途**: 源码引用、交叉引用、验证报告、维护指南、质量保证

---

## 文件清单

| 文件 | 用途 | 更新频率 |
|:---|:---|:---:|
| `SOURCE_REFERENCE_INDEX.md` | 所有 GitHub 源码引用的集中索引 | 版本发布时 |
| `CROSS_REFERENCE_INDEX.md` | 文档间交叉引用与推荐阅读路径 | 新增文档时 |
| `VALIDATION_REPORT.md` | Phase 3 验证结果 (T11-T14) | 验证后 |
| `GLOSSARY_SUPPLEMENT.md` | 21-25 新文档专用术语表 | 术语变更时 |
| `COMPLETION_REPORT.md` | 项目完成度里程碑报告 | 里程碑时 |
| `EXECUTION_SUMMARY.md` | 计划与实际执行对照 | 阶段完成时 |
| `MAINTENANCE_GUIDE.md` | 长期维护操作指南 | 按需 |
| `VERSION_TRACKING_TEMPLATE.md` | 月度版本跟踪模板 | 每月 |
| `ADVANCED_ENTRY_GUIDE.md` | 按角色分类的快速入口 | 按需 |
| `QUALITY_ASSURANCE_REPORT.md` | 质量保证检查结果 | QA 后 |
| `PLAN_SVELTE_SIGNALS_COMPREHENSIVE_ANALYSIS.md` | 原始执行计划 | 计划变更时 |

---

## 使用方式

1. **查找源码引用**: 查阅 `SOURCE_REFERENCE_INDEX.md`
2. **按主题阅读**: 查阅 `CROSS_REFERENCE_INDEX.md` 的推荐阅读路径
3. **验证内容准确性**: 查阅 `VALIDATION_REPORT.md`
4. **术语疑问**: 查阅 `GLOSSARY_SUPPLEMENT.md`
5. **维护操作**: 查阅 `MAINTENANCE_GUIDE.md`
